五、API 封装规则（固定）
--------------

javascript

// utils/request.js
import axios from 'axios'
import { ElMessage } from 'element-plus'
import { useUserStore } from '@/stores/user'
const request = axios.create({
  baseURL: 'http://localhost:8080',
  timeout: 10000
})
// 请求拦截器：自动添加 Token
request.interceptors.request.use(config => {
  const store = useUserStore()
  if (store.token) {
    config.headers.Authorization = `Bearer ${store.token}`
  }
  return config
})
// 响应拦截器：统一处理 code
request.interceptors.response.use(
  response => {
    const res = response.data
    if (res.code === 200) {
      return res.data  // 直接返回 data 内容
    } else if (res.code === 401) {
      ElMessage.error('登录已过期，请重新登录')
      const store = useUserStore()
      store.logout()
      window.location.href = '/login'
      return Promise.reject(res)
    } else {
      ElMessage.error(res.message || '请求失败')
      return Promise.reject(res)
    }
  },
  error => {
    ElMessage.error('网络错误，请稍后重试')
    return Promise.reject(error)
  }
)
export default request

六、Token 存储规则（固定）
----------------

* 登录成功后，Token 存入 **localStorage**（键名：`token`）

* 用户信息存入 **localStorage**（键名：`user`）

* Pinia 初始化时从 localStorage 读取

* 退出登录时清除 localStorage

七、具体页面功能要求（按角色）
---------------

### 7.1 登录页（/login）

* 两个输入框：用户名、密码

* 一个登录按钮

* 登录成功 → 跳转到 `/dashboard`

* 登录失败 → ElMessage 显示后端返回的错误信息

### 7.2 系统管理员页面

**所有列表页面统一要求：**

* 表格展示数据

* 顶部有「新增」按钮 → 弹出 Dialog 表单

* 每行有「编辑」「删除」按钮

* 分页组件（每页 10 条）

具体页面：

* `/admin/students`：学生列表（学号、姓名、学院、操作）

* `/admin/teachers`：教师列表（工号、姓名、学院、操作）

* `/admin/courses`：课程列表（名称、容量、教师、学院、操作）

* `/admin/enrollments`：选课列表（学生、课程、状态、操作）

* `/admin/grades`：成绩录入（选课记录 ID、分数、提交按钮）

* `/admin/chart`：柱状图（横轴课程名，纵轴选课人数），显示 PNG 下载按钮

### 7.3 学院管理员页面

与系统管理员类似，但操作范围仅限于本学院。

* 抽签页面（`/dept/draw`）：选择课程 → 点击「抽签」按钮 → 二次确认 → 显示中签/未中签名单

### 7.4 教师页面

* `/teacher/courses`：查看自己的课程列表，可编辑课程名称和介绍

* `/teacher/courseware/:courseId`：上传文件（input type="file"）+ 查看已上传文件列表

* `/teacher/prerequisite/:courseId`：下拉选择先修课程 + 提交按钮（可设为 null）

* `/teacher/selected`：查看自己所有课程中签学生列表

* `/teacher/grades`：录入成绩（选择课程 → 显示该课程已中签学生 → 输入分数 → 提交）

* `/teacher/stats/:courseId`：显示柱状图（分数段人数）+ 及格率

* `/teacher/messages`：左侧显示会话列表，右侧显示聊天记录 + 输入框发送

### 7.5 学生页面

* `/student/enroll`：浏览全部课程 → 点击「选课」按钮 → 检查先修课程 → 成功或提示错误

* `/student/my-courses`：已选课程列表（显示状态：待抽签/中签/未中签）

* `/student/info`：显示个人基本信息（用户名、学号、姓名、学院）

* `/student/grades`：仅显示已中签且有成绩的课程

* `/student/courseware/:courseId`：显示该课程的所有课件 → 点击「下载」按钮下载文件

* `/student/messages`：同教师消息界面

* `/student/draw-result`：显示自己所有选课的抽签结果列表

九、路由守卫（固定）
----------

javascript

// router/index.js
router.beforeEach((to, from, next) => {
  const store = useUserStore()
  const token = store.token || localStorage.getItem('token')

  if (to.path === '/login') {
    next()
  } else {
    if (token) {
      // 检查角色权限，无权限则跳转到 403 或首页
      next()
    } else {
      next('/login')
    }
  }
})
